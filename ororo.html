Сегодня мы рассмотрим замечательный CSS препроцессор - SASS. Сам я подсел на него не так давно, но, кажется, надолго. Но сначала мы попытаемся понять что такое препроцессор и для чего он нужен.

Препроцессор — это компьютерная программа, принимающая данные на входе и выдающая данные, предназначенные для входа другой программы (например, компилятора). О данных на выходе препроцессора говорят, что они находятся в препроцессированной форме, пригодной для обработки последующими программами (компилятор). Результат и вид обработки зависят от вида препроцессора; так, некоторые препроцессоры могут только выполнить простую текстовую подстановку, другие способны по возможностям сравниться с языками программирования.

Конкретно в CSS препроцессор является неким синтаксическим сахаром, упрощающим написание кода, избавляющим от рутины и несколько расширяющим этот самый CSS.

Пример из SASS: 
$colorBG: #d1d1d1;
$colorTXT: #1e1e1e;
%block($widht) {
	background: $colorBG;	
	color: $colorTXT;
	width: $width; 
} 

.block1 {
	@extend %block('300px');
}
.block2 {
	@extend %block('500px');
}
.block3 {
	@extend %block('800px');
}
Компелируется в: 
.block1 {
	background: #d1d1d1;
	color: #1e1e1e;
	width: 300px;
}
.block2 {
	background: #d1d1d1;
	color: #1e1e1e;
	width: 500px;
}
.block3 {
	background: #d1d1d1;
	color: #1e1e1e;
	width: 800px;
}

В данном примере мы видим всю суть препроцессоров. Переменные, инкапсуляции, миксины это то, за что их так любят верстальщики. Возможно, данный пример вас и не впечетлил, но если представить, что подобных блоков будет не 3, а 300 то всё становится на свои места.

Использование препроцессора экономит кучу времени на написании кода и последующей отладки. Ведь для того, чтобы исправить цвет всех схожих блоков не нужно будет менять каждую строку отдельно, хватит лишь заменить цвет в переменной и препроцессор сделает всю работу за вас.

Из CSS препроцессоров самыми популярными на данный момент являются LESS, SASS и Stylus.

Данная статья посвящена именно SASS, так как свою первую статью я хотел бы написать о том, что знаю и использую в каждом проекте.

Так вот, по словам создателей, Sass является наиболее развитым, стабильным и мощным расширением языка CSS в мире.

Язык SASS имеет два синтаксиса:
  • Непосредственно SASS - отличается отсутсвием фигурных скобок. Уровень вложенности обозначается отступами.
  • SCSS(Sassy CSS) - использует фигурные скобки, как и CSS.

Любители простого, минималистического кода (как в Ruby, Coffeescript) могут найти интересным использование первого синтаксиса, я же использую именно второй. Этот выбор я сделал, потмоу, что второй синтаксис является обратно-совместимым. Это значит, что нативный CSS может прекрасно использоваться как SCSS, но не как SASS.

Особенности языка.

Первой и главной особенностью языка SASS являются вложенные селекторы. Это избавляет вас от написания излишнего кода.

.block {
	color: #111;
	widht: 500px;
	margin: 0 auto;
}
.block ul {
	font-size:0;
}
.block ul li {
	font-size: 14px;
	display: inline-block;
}

Можно записать так:

.block {
	color: #111;
	widht: 500px;
	margin: 0 auto;

	ul {
		font-size: 0;

		li {
			font-size: 14px;
			display: inline-block;
		} 
	}
}

Так, как я именую классы по методологии БЭМ я использую SCSS и в этих целях. Получается очень удобно. Из этого:

.block {
	font-size: 10px;
	&__inner {
		max-width: 1200px;
	}
	$__outer {
		padding: 0 15px;
	}
}

Получается это:

.block {
	font-size: 10px;
}
.block__inner {
	max-width: 1200px;
}
.block__outer {
	padding: 0 15px;
}
